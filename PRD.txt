PRD: 模块化旅游日程规划工具 (YoteiTrip MVP)
1. 产品愿景
打造一个去中心化、非线性的旅游日程工具。用户无需预设目的地和天数，而是通过“积木式”的模块搭建，实现从第一天起点的分钟级精确规划。

2. 核心交互逻辑
非问答式启动：首页即是日程表，默认从“Day 1”开始。

无限扩展性：点击右上角 + 号动态增加天数。

横向视图：采用左右滑动（Mobile）或 Tab 切换（Web）来浏览不同日期。

纵向时间轴：每一天内部是垂直排列的模块化卡片。

3. 功能模块详细说明
A. 日程架构 (The Itinerary Stack)
每个日程由以下四类“积木模块”组成：

Transport (运输)：机场/车站。允许设置自定义起始时间（解决下午/晚上到达的情况）。

Stay (停留)：景点、活动、购物点。

Meal (餐饮)：午餐、午茶、晚餐区域。

Accommodation (住宿)：酒店位置。支持“同前一晚”或“新区域”快捷选项。

B. Google Maps API 集成（核心引擎）
地点检索：所有模块的地点输入均对接 Places Autocomplete。

智能通勤计算：

自动提取相邻两模块的 place_id。

调用 Distance Matrix API 计算实时/预估通勤时长。

跳转外链：点击通勤模块直接唤起 Google Maps 原生 App 的导航界面。

时间推算逻辑：

当前模块结束时间 = 开始时间 + 用户手动选择的停留时长。

下一模块开始时间 = 当前模块结束时间 + Google 计算的通勤时间。

4. 视觉与 UI 规范
风格：极简模块化（Modular Card Design）。

双端适配：

Mobile：支持手势左右滑动切换天数。

Web：中心化窄屏容器，保持 App 级的一致观感。

状态反馈：通勤模块显示为两个卡片之间的连接线，并标注“步行/驾车/公交时间”。

5. 开发阶段规划 (Phased Instructions)
Phase 1: Zustand 状态树搭建（支持多天与多模块数据存储）。

Phase 2: 核心 UI 开发（卡片渲染、横向切换逻辑）。

Phase 3: Google Maps API 联调（地点选择与时间推算）。

Phase 4: 优化时间轴顺延逻辑。
